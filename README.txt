DOCKER:

Docker Hub:
1- Pull from anasaleid/anas_aleid_hw4

Running locally:
1- clone my bitbucket repo
2- import project using intellij. Make sure to select maven as the import method. Don't import the entire folder
that you cloned. The JavaOpenChess folder should be the folder you import into intellij. 
3- From there, you can right click on the "chessApplication" class and select run from the dropdown in 
order to start the spring application.

Application playing against itself or you:
If you want two CPUs playing vs each other, there is a main method in the jChessApp that is commented out.
You can uncomment the method along with one of the three game modes within the main method.
1- CPUvsCPU- will run a game with two CPUs vs each other. It will end the game no matter what after 300 moves
since they it usually ends in a draw with two kings moving around on each side. This method will sometimes
cause an error due to some random number generator logic that was in the game already. If that happens, just
rerun the game. It doesn't happen too often.
2- HumanVsCpu(false)- this will start a game with you playing as the white pieces.
3- HumanVsCpu(true)- this will start a game with you playing as the black pieces.

Running from docker locally:
1- The docker video shows how to run the docker image locally. If you'd like to grab the docker image
from dock hub instead, then run the "docker pull anasaleid/anas_aleid_hw4" command. I made the docker hub
repo public so you should be able to pull from it if you don't want to build the image yourself.
2- You should be able to run the "docker run -p 8080:8080 anasaleid/anas_aleid_hw4" command right after
in order to start the application.

Running from docker on AWS:
1- Refer to the video: https://youtu.be/9Hzgfffw-k4

CAPSTAN:
I spent the majority of this project trying to figure out how to get capstan to work. Initially I tried using capstan straight from windows,
but I figured out that you can not build capstan images from windows due to what I believe is hypervisor issues. I then tried capstan from
a linux VM. I still ran into a ton of issues. I found out I had to edit the VM processor settings to allow KVM to work properly. I then started running into issues were if I try to do capstan run "myjar.jar" it uses an old version of java that does not support the spring boot
class I was using in my project. I then created a capstan file to try to change the base that is used in the OSv. I started running into the
issue were doing capstan build does not copy over my jar file to the OSv even though I included the necessary command in the capstan file. I
figured that if I got docker working perfectly then I shouldn't worry too much with regards to loosing a lot of points for this assignment, but thats for you to decide. I am still confident that my program will still work with a working capstan instance, so I will still include the capstan file I was using.

In order to build and run a capstan image:
1- open a terminal within the JavaOpenChess directory
2- run the command "capstan build -v"
3- run the command "capstan run -f 8080:7070 -v"
4- I believe that capstan uses the IP "192.168.122.15", but like I said before I could never get to this point so I am not 100% sure.
5- You should be able to interact with the chess API using that IP and the port 8080 just like with the docker.
